## Changes Made

### 1. **utils/ai_engine.py** - Added AI Question Generators

#### New Functions Added:

##### `generate_mcq_questions(role: str, n: int = 10) -> list`
- **Purpose**: Generate multiple-choice questions using AI
- **Input**: Role/skill name (e.g., "Frontend Developer", "DevOps Engineer") 
- **Output**: List of MCQ dicts with: `q`, `options` (4 items), `answer` (0-3 index)
- **Features**:
  - Role-specific prompts that generate relevant questions
  - Robust JSON parsing with multiple fallback formats
  - Validates each question has proper structure
  - Returns empty list only if truly unavailable
- **Used by**: Screening Level 1 (MCQ Round)

##### `generate_coding_questions(role: str, n: int = 2) -> list`
- **Purpose**: Generate coding challenges using AI
- **Input**: Role/skill name
- **Output**: List of coding problem dicts with: `id`, `func`, `title`, `difficulty`, `description`, `starter`, `test_cases`
- **Features**:
  - Generates appropriate difficulty levels
  - Creates valid Python function stubs
  - Includes test cases with expected outputs
  - Validates all required fields present
  - Works for any role (not limited to predefined list)
- **Used by**: Screening Level 2 (Coding Round)

#### Improvements to Existing Functions:
- `generate_question()` - Already AI-powered, now confirmed working for all roles
- `_ollama()` - Connection handler to Ollama AI server (no changes needed)
- All functions have proper error logging for debugging

### 2. **screening/screening_routes.py** - Updated to Use AI

#### Changes to Level 1 Route:

## Troubleshooting

### Problem: "AI unavailable – start Ollama: `ollama serve`"
**Solution**: 
1. Open a terminal
2. Run: `ollama serve`
3. Keep it running in background

### Problem: Empty questions on non-standard roles
**Solution**: 
1. Check Ollama is running
2. Run: `python3 VERIFY_AI_SETUP.py`
3. Check logs for JSON parsing errors

### Problem: Questions too generic
**Solution**: 
- This shouldn't happen with new AI generation
- If it does, check Ollama model is properly configured
- Try: `ollama pull llama3`

### Problem: Slow question generation
**Solution**: 
- Ollama first run is slow (model loading)
- Subsequent generations are faster
- Check Ollama system requirements
